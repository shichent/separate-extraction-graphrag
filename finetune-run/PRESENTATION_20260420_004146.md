# NER Fine-Tune Transfer Study — Chinese Novels

_Shichen Tang, Houbo He · COMP 584 · compiled 2026-04-20 11:21_

## Research question

Does fine-tuning on **general-domain Chinese NER** transfer to **Chinese fantasy/cultivation novels**, and does combining multiple general NER corpora help?

## Setup

- **Base model:** `unsloth/Qwen3.5-9B` (9.5 B params)
- **Fine-tune method:** LoRA, r=32, all linear layers, bf16, 3 epochs, 10k samples/dataset
- **Hardware:** 3× NVIDIA L40 (48 GB each)
- **Training datasets:**
  - `whoisjones/fiNERweb` (cmn) — char-span, free-form fine-grained labels
  - `msra_ner` — token-BIO, {PER, ORG, LOC}
  - `nlhappy/CLUE-NER` — 10 CLUENER categories
  - `combined` — equal mix (3,333 each) of the three above
- **Target novels:**
  - `1.txt` — 《苟在初圣魔门当人材》 (~2.1 MB, modern cultivation novel)
  - `2.txt` — 《凡人修仙传》 (~2.6 MB, modern cultivation novel)

## Takeaways

1. **Fine-tuning helps closed-vocabulary schemas most.** On `msra`, specialist F1 rose **+31.8 pp** over the zero-shot Qwen3.5-9B baseline (58.48% → 90.25%).
2. **Fine-tuning can hurt open-vocabulary schemas.** On `fiNERweb` (~50 fine-grained labels), specialist F1 actually dropped **-4.7 pp** vs zero-shot (60.33% → 55.64%) — 10k fine-tune samples over-fit without covering the label space.
3. **Combined adapter underperforms the specialists on average** (mean F1 68.21% vs 75.40%). It only wins on fiNERweb, where the extra corpora compensate for the sparse label space.
4. **On the target novels**, `fiNERweb` extracts the most entities overall (9,425 across both novels).
5. fiNERweb's open-vocabulary labels (e.g. `person / fictional character`, `cultural reference / artifact`) naturally align with fantasy fiction, while MSRA's closed `{PER, ORG, LOC}` produces cleaner but coarser labels.
6. CLUENER's news-domain schema (`address`, `scene`, `position`) mislabels sect names and martial-arts techniques — strong extraction, wrong ontology.
7. **Label-precision spikes hardest on closed schemas.** Zero-shot LP is only 16–52%, meaning the base model finds entity spans but guesses the wrong label string. Fine-tuning on MSRA raises LP from 52% → 93%.
